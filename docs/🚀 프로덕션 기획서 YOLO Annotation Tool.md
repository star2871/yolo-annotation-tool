# 🚀 프로덕션 기획서: YOLO Annotation Tool

## 1. 앱 이름

- **YoloTrace (욜로트레이스)**
  - 의미: YOLO 모델을 위한 데이터를 정밀하게 추적(Trace)하고 라벨링한다는 직관적인 의미를 담았습니다.

## 2. 앱의 핵심 목표

- AI 모델 학습을 위한 데이터셋 구축 과정을 자동화하여, 연구 및 개발자가 고품질 라벨 데이터를 신속하게 확보할 수 있도록 돕는 것을 목표로 합니다.   

## 3. 해결하고자 하는 문제

- 

  **데이터셋 구축의 비효율성:** 수많은 이미지를 수동으로 라벨링하는 과정에서 발생하는 시간 소모와 작업 피로도를 해결합니다.   

- 

  **좌표 정규화 및 포맷 오류:** 수동 라벨링 시 흔히 발생하는 YOLO 표준 포맷(정규화 좌표) 계산 오류를 방지합니다.   

- 

  **워크플로우 파편화:** 이미지 탐색, 라벨링, 저장, 검증이 분리되어 있는 기존 툴의 파편화된 흐름을 통합하여 작업 연속성을 보장합니다.   

## 4. 타겟 사용자

- **AI/ML 모델 연구원:** 객체 탐지 모델 학습을 위해 대량의 이미지 데이터를 정제해야 하는 전문가.
- **데이터 분석가 및 개발자:** LLM 및 컴퓨터 비전 모델 개발 과정에서 직접 데이터셋을 구축하는 실무자.
- **컴퓨터 비전 입문자:** 학습용 데이터셋 구조를 이해하고 빠르게 라벨링 실습을 하고자 하는 학생 및 엔지니어.

## 5. 핵심 구현 기능 (MVP)

- 

  **데이터 자동화:** 로컬 이미지 파일 자동 스캔 및 라벨 파일 1:1 매칭 생성.   

- 

  **인터랙티브 라벨링:** 스냅형 박스 툴과 반투명 오버레이를 활용한 정밀 라벨링 인터페이스.   

- 

  **워크플로우 가속:** 단축키(Enter, Ctrl+Z, Delete) 기반의 자동 저장 및 다음 이미지 이동 시스템.   

- 

  **좌표 규격화:** 마우스 절대 좌표를 YOLO 표준 정규화 포맷($x_{center}, y_{center}, width, height$)으로 자동 변환.   

## 6. 사용자 경험(UX) 원칙

- 

  **Flow-Centric Design:** 마우스와 키보드의 시선 이동을 최소화하여 라벨링 작업의 '흐름'이 끊기지 않도록 설계합니다.   

- 

  **Minimalist Feedback:** 시스템 상태(저장 여부, 라벨링 완료 등)를 즉각적이고 직관적인 시각적 신호(색상 변경, 아이콘)로 전달합니다.   

- 

  **Zero-friction Workflow:** 사용자가 설정에 신경 쓰지 않아도 작업만 하면 데이터가 규격에 맞게 자동 처리되는 환경을 제공합니다.   

## 7. 기술 스택 요약

- 

  **프론트엔드:** React.js (Vite), Tailwind CSS, Zustand, LocalStorage   

- 

  **백엔드 (로컬 브릿지):** Node.js (Express.js), `fs` 모듈   

- 

  **데이터 관리:** 파일 기반 로컬 관리 (이미지 폴더 및 라벨 폴더 1:1 매핑)   

## 8. MVP 개발 일정표

| **단계**    | **주요 작업 내용**                                        | **예상 기간** |
| ----------- | --------------------------------------------------------- | ------------- |
| **Phase 1** | 로컬 파일 시스템 연동 및 이미지 로딩 로직 구현            | 1주차         |
| **Phase 2** | Canvas 내 바운딩 박스 생성 및 YOLO 좌표 변환 엔진 구축    | 2주차         |
| **Phase 3** | 단축키 기반 자동 저장 및 다음 이미지 전환 워크플로우 완성 | 3주차         |
| **Phase 4** | 검증 모드 및 최종 인터페이스 폴리싱                       | 4주차         |
