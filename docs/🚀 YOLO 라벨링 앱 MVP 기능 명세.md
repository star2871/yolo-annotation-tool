# 🚀 YOLO 라벨링 앱 MVP 기능 명세

본 문서는 YOLO 모델 학습을 위한 데이터셋 구축 효율화를 목표로 하는 MVP 기능 목록입니다.

## 1. 핵심 데이터 파이프라인

- **자동 파일 스캔:** `images/` 폴더 내 이미지 파일을 감지하여 작업 대기열 생성.
- **1:1 매핑:** 이미지 로드 시 동일한 이름의 `.txt` 라벨 파일(`labels/` 폴더) 자동 생성.
- **원본 보호(Read-only):** 원본 이미지 파일의 수정/삭제를 차단하고 라벨 데이터만 독립적으로 관리.

## 2. 인터랙티브 라벨링

- **스냅형 박스 툴:** 마우스 드래그 시 이미지 경계에 정확히 맞추는 가이드라인 제공.
- **시각화 레이어:** 반투명 오버레이를 통해 라벨링 영역을 실시간 확인.
- **피드백 시스템:** 박스 완료 시 테두리 색상 변경을 통해 상태 전달.

## 3. 워크플로우 자동화

- **비동기 자동 저장:** 박스 생성 완료 즉시 데이터가 파일에 자동 기록.
- **단축키 기반 조작:**
  - `Enter`: 박스 확정, 파일 저장 후 다음 이미지로 자동 로딩.
  - `Ctrl+Z`: 마지막 작업 취소(Undo).
  - `Delete`: 선택된 박스 삭제.
- **검증 모드:** 저장된 라벨을 즉시 이미지 위에 오버레이하여 데이터 오염 여부 확인.

## 4. 데이터 규격화

- **자동 정규화:** 마우스 좌표 $(x, y)$를 YOLO 학습 포맷($<class\_id> \ x\_center \ y\_center \ width \ height$)으로 자동 변환.

### 💡 MVP 개발 우선순위 로드맵

1. **[기초]** 이미지 로딩 및 마우스 좌표 기반 바운딩 박스 그리기 구현
2. **[저장]** YOLO 포맷 텍스트 파일 생성 및 폴더 매핑 구현
3. **[자동화]** 단축키 기반 다음 이미지 이동 및 비동기 저장 로직 적용

*본 MVP는 '라벨링 품질'과 '작업 속도'를 최우선으로 고려하여 설계되었습니다.*
