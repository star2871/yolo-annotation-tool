욜로모델에 사용할 이미지 labelling 할 수 있는 앱
생성형 AI와의 전체 대화 내용

욜로모델에 사용할 이미지에 boxplot을 선택하면 boxplot의 위치가 labels에 그 이미지에 해당하는 좌표를 저장해서 욜로모델이 사용할 수있는 앱을 만들려고해 이 앱의 MVP 기능 목록을 정의해 줘. 각 기능은 매우 구체적이고 명확하게 설명할 수 있어야 해. 추상적으로 접근하지 않았으면 좋겠어.
[작업 지침]
-	MVP 관점에서 '있어서는 안 될' 기능만 포함시켜 줘. (예: 꾸미기 기능, 통계 기능 등은 제외)
-	라벨링 기술에 집중해줘(이미지의 부분을 선택하면 그부분을 labels 라는 폴더에 저장되게 해줘)

이미지 라벨링 앱에 몇 가지 의견이 있어. 아래에 의견을 정리해놓을 테니까, 읽어보고 무조건 긍정만 하지 말고 비판적 시선으로 내 의견을 검토해 봐. 어떤 의견이 가장 이상적인 아이디어인지 냉철하게 판단하고 생산적인 의견을 제시해 줘. 만약 창의적이고 독특한 아이디어가 있다면 3가지 정도 제안해 줘.
[나의 의견]
1.	박스플롯을 그릴때 내가 구별하기 쉽게 그리기 쉽게 햇으면 좋겟어
2.	이미지를 넣으면 자동으로 그 이미지가 저장되고 그 이미지를 바탕으로 박스플롯의 위치를 labels라는 폴더에 넣어주면 좋겟어 완성된 이미지하고 라벨을 여러가지이미
3.	실시간으로 저장 됬으면 좋겟어

[최종 기술 스택 목록]
1. 프론트엔드 (React Web)
React.js (Vite): UI 렌더링 및 인터랙티브 라벨링 로직 구현.

Tailwind CSS: 최소한의 코드로 구현하는 빠르고 직관적인 UI 디자인.

Zustand: 현재 이미지 상태, 바운딩 박스 리스트 등 전역 상태 관리.

LocalStorage: 사용자 환경 설정(클래스 목록 등) 및 임시 세션 메타데이터 저장.

2. 백엔드 (Node.js Bridge)
Node.js (Express.js): 브라우저가 직접 접근할 수 없는 로컬 파일 시스템(fs 모듈)과의 통신 담당.

fs (File System) 모듈: .txt 라벨 파일 생성, 수정 및 labels/ 폴더 내 데이터 저장 전용.

3. 데이터 구조
파일 기반 관리: 별도의 데이터베이스(Firestore 등) 없이 images/와 labels/ 폴더의 1:1 매칭 구조를 통한 로컬 데이터 저장.
