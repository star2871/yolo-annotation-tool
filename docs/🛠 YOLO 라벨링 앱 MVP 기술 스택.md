# 🛠 YOLO 라벨링 앱 MVP 기술 스택

본 기술 스택은 **로컬 파일 처리 속도**와 **개발 생산성**에 최적화되어 있습니다.

### 1. 프론트엔드 (React Web)

- **React.js (Vite):** UI 렌더링 및 인터랙티브 라벨링 로직 구현.
- **Tailwind CSS:** 최소한의 코드로 구현하는 빠르고 직관적인 UI 디자인.
- **Zustand:** 현재 이미지 상태, 바운딩 박스 리스트 등 전역 상태 관리.
- **LocalStorage:** 사용자 환경 설정(클래스 목록 등) 및 임시 세션 메타데이터 저장.

### 2. 백엔드 (Node.js Bridge)

- **Node.js (Express.js):** 브라우저가 직접 접근할 수 없는 로컬 파일 시스템(`fs` 모듈)과의 통신 담당.
- **`fs` (File System) 모듈:** `.txt` 라벨 파일 생성, 수정 및 `labels/` 폴더 내 데이터 저장 전용.

### 3. 데이터 구조

- **파일 기반 관리:** 별도의 데이터베이스 없이 `images/`와 `labels/` 폴더의 1:1 매칭 구조를 통한 로컬 데이터 저장.

### 💡 핵심 요약

- **서버는 로컬 전용:** 백엔드는 클라우드와 통신하지 않고, 오직 PC 내 특정 폴더의 파일만 읽고 쓰는 '로컬 파일 제어기'로 사용합니다.
- **상태 관리는 프론트엔드:** 앱의 메인 로직(좌표 계산, 박스 생성 등)은 리액트 내에서 처리하며, 결과물만 백엔드를 통해 로컬 파일로 저장합니다.
- **데이터 무결성:** 로컬 파일 시스템을 직접 사용하므로 네트워크 에러가 없으며, 데이터가 파일 형태로 즉시 저장되어 즉시 모델 학습에 활용 가능합니다.

이제 이 구조를 기반으로 프로젝트를 시작할 준비가 되었습니다. 가장 먼저 구현해야 할 로직(예: 이미지 로드 로직, 파일 저장 API, 또는 바운딩 박스 계산 로직)이 무엇인지 알려주시면 구체적인 코드를 제안해 드리겠습니다.
