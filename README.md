# 🚀 YoloTrace (욜로트레이스)

> **AI 객체 탐지 모델 학습을 위한 고속 & 정밀 YOLO Annotation Tool**

YoloTrace는 YOLO 모델 학습을 위한 데이터셋 구축 과정을 효율화하고 자동화하여, 연구원 및 개발자가 고품질 라벨 데이터를 신속하게 확보할 수 있도록 돕는 로컬 기반 라벨링 웹 애플리케이션입니다.

![YOLO 라벨링 앱 워크플로우](./docs/YOLO%20라벨링%20앱의%20흐름도%20와일드%20프레임.png)

---

## 🎯 기획 배경 및 해결하고자 하는 문제

1. **데이터셋 구축의 비효율성 해결:** 수많은 이미지를 수동으로 라벨링하는 과정에서 발생하는 시간 소모와 작업 피로도를 대폭 줄였습니다.
2. **정확한 포맷 변환 보장:** 수동 라벨링 시 흔히 발생하는 YOLO 표준 포맷(정규화 좌표) 계산 오류를 방지하고 자동 변환합니다.
3. **워크플로우 파편화 통합:** 이미지 탐색, 라벨링, 저장, 검증이 분리되어 있던 기존 툴의 파편화된 흐름을 통합하여 끊김 없는 작업 연속성을 제공합니다.

---

## ✨ 핵심 기능 (Key Features)

### 1. 데이터 파이프라인 자동화
- **자동 파일 스캔:** 로컬 `images/` 폴더 내 이미지 파일을 자동 감지하여 작업 대기열을 생성합니다.
- **1:1 자동 매핑:** 이미지 로드 시 동일한 이름의 `.txt` 라벨 파일을 `labels/` 폴더에 생성 및 매핑합니다.
- **원본 보호 (Read-only):** 원본 이미지 파일의 수정 및 삭제를 차단하고 라벨 데이터만 독립적으로 안전하게 관리합니다.

### 2. 인터랙티브 라벨링 (Interactive Labeling)
- **스냅형 박스 툴:** 마우스 드래그 시 이미지 경계에 정확히 맞출 수 있는 가이드라인을 제공합니다.
- **시각화 레이어:** 반투명 오버레이를 통해 라벨링 영역을 실시간으로 확인하고 정밀도를 높입니다.
- **피드백 시스템:** 박스 완성 시 테두리 색상 변경 등 즉각적인 시각적 피드백을 제공하여 작업 상태를 명확히 전달합니다.

### 3. 워크플로우 가속화 (Zero-friction Workflow)
- **비동기 자동 저장:** 바운딩 박스 생성 완료 즉시 데이터가 로컬 파일에 자동 기록됩니다.
- **단축키 기반 고속 조작:**
  - `[Enter]`: 박스 확정 및 파일 저장 후 다음 이미지로 자동 로딩
  - `[Ctrl+Z]`: 마지막 작업 취소 (Undo)
  - `[Delete]`: 선택된 바운딩 박스 삭제
- **검증 모드:** 저장된 라벨을 즉시 이미지 위에 오버레이하여 데이터 오염 및 정확도를 즉시 확인 가능합니다.

### 4. 데이터 규격화 (Standardization)
- **자동 정규화 엔진:** 사용자의 마우스 절대 좌표 $(x, y)$를 YOLO 학습 표준 포맷으로 실시간 자동 변환합니다.
  - `Format: <class_id> <x_center> <y_center> <width> <height>`

## 🎨 UI/UX 디자인 시스템 (M3 Based)

작업자의 눈 피로도를 최소화하고, 작업 영역의 가시성을 극대화하기 위해 **다크 모드(Dark Mode)** 기반의 M3 디자인 시스템을 적용했습니다.

### 디자인 원칙
- **Focus First:** 작업 영역(이미지 캔버스)의 시각적 노이즈 최소화
- **Efficient Interaction:** 마우스 이동 경로 최적화를 통한 3단 레이아웃 내 작업 완결
- **Contextual Clarity:** 현재 상태를 즉각적으로 알 수 있는 고대비 컬러 피드백

### 3단 레이아웃 구성
1. **좌측 (Sidebar):** 이미지 리스트 및 라벨링 완료 상태 표시 (`FileList`)
2. **중앙 (Canvas):** 마우스 드래그 박스 툴 및 정규화 좌표 실시간 피드백 오버레이 (`InteractionCanvas`)
3. **우측 (Panel):** 클래스 선택, 라벨 리스트 아이템 수정/삭제, 단축키 가이드 (`ClassController`)

## 🛠 기술 스택 (Tech Stack)

로컬 파일 처리 속도와 개발 생산성에 최적화된 아키텍처를 채택했습니다. 별도의 데이터베이스 없이 로컬 파일 시스템을 직접 제어하여 무결성과 속도를 확보했습니다.

### Frontend
- **React.js (Vite):** 빠르고 부드러운 UI 렌더링 및 인터랙티브 라벨링 로직 구현
- **Tailwind CSS:** 디자인 시스템을 반영한 빠르고 직관적인 UI 스타일링
- **Zustand:** 이미지 상태, 바운딩 박스 리스트 등 전역 상태의 효율적 관리
- **LocalStorage:** 사용자 환경 설정(클래스 목록 등) 및 임시 세션 데이터 보관

### Backend (Local Bridge)
- **Node.js (Express.js):** 브라우저가 직접 접근할 수 없는 로컬 파일 시스템과의 브릿지 역할 담당
- **`fs` 모듈:** `.txt` 라벨 파일의 빠른 생성, 읽기, 쓰기 등 로컬 데이터 저장 전용 통신망
