# 🚀 YoloTrace (욜로트레이스)

> **AI 객체 탐지 모델 학습을 위한 고속 & 정밀 YOLO Annotation Tool**

YoloTrace는 YOLO 모델 학습을 위한 데이터셋 구축 과정을 효율화하고 자동화하여, 연구원 및 개발자가 고품질 라벨 데이터를 신속하게 확보할 수 있도록 돕는 로컬 기반 라벨링 웹 애플리케이션입니다.

![YOLO 라벨링 앱 워크플로우](./docs/YOLO%20라벨링%20앱의%20흐름도%20와일드%20프레임.png)

---

## 🎯 기획 배경 및 해결하고자 하는 문제

1. **데이터셋 구축의 비효율성 해결:** 수많은 이미지를 수동으로 라벨링하는 과정에서 발생하는 시간 소모와 작업 피로도를 대폭 줄였습니다.
2. **정확한 포맷 변환 보장:** 수동 라벨링 시 흔히 발생하는 YOLO 표준 포맷(정규화 좌표) 계산 오류를 방지하고 자동 변환합니다.
3. **워크플로우 파편화 통합:** 이미지 탐색, 라벨링, 저장, 검증이 분리되어 있던 기존 툴의 파편화된 흐름을 통합하여 끊김 없는 작업 연속성을 제공합니다.

---

## ✨ 핵심 기능 (Key Features)

### 1. 데이터 파이프라인 자동화
- **자동 파일 스캔:** 로컬 `images/` 폴더 내 이미지 파일을 자동 감지하여 작업 대기열을 생성합니다.
- **1:1 자동 매핑:** 이미지 로드 시 동일한 이름의 `.txt` 라벨 파일을 `labels/` 폴더에 생성 및 매핑합니다.
- **원본 보호 (Read-only):** 원본 이미지 파일의 수정 및 삭제를 차단하고 라벨 데이터만 독립적으로 안전하게 관리합니다.

### 2. 인터랙티브 라벨링 (Interactive Labeling)
- **스냅형 박스 툴:** 마우스 드래그 시 이미지 경계에 정확히 맞출 수 있는 가이드라인을 제공합니다.
- **시각화 레이어:** 반투명 오버레이를 통해 라벨링 영역을 실시간으로 확인하고 정밀도를 높입니다.
- **피드백 시스템:** 박스 완성 시 테두리 색상 변경 등 즉각적인 시각적 피드백을 제공하여 작업 상태를 명확히 전달합니다.

### 3. 워크플로우 가속화 (Zero-friction Workflow)
- **비동기 자동 저장:** 바운딩 박스 생성 완료 즉시 데이터가 로컬 파일에 자동 기록됩니다.
- **단축키 기반 고속 조작:**
  - `[Enter]`: 박스 확정 및 파일 저장 후 다음 이미지로 자동 로딩
  - `[Ctrl+Z]`: 마지막 작업 취소 (Undo)
  - `[Delete]`: 선택된 바운딩 박스 삭제
- **검증 모드:** 저장된 라벨을 즉시 이미지 위에 오버레이하여 데이터 오염 및 정확도를 즉시 확인 가능합니다.

### 4. 데이터 규격화 (Standardization)
- **자동 정규화 엔진:** 사용자의 마우스 절대 좌표를 YOLO 학습 표준 포맷으로 실시간 자동 변환합니다.
  - `Format: <class_id> <x_center> <y_center> <width> <height>`

### 5. 🤖 AI 자동 라벨링 (Zero-Shot Auto Labeling) `NEW`
- **OWL-ViT 기반 Zero-Shot 탐지:** `Xenova/owlvit-base-patch32` 모델을 백엔드에서 구동하여, 별도 학습(fine-tuning) 없이 자연어 텍스트만으로 객체를 탐지합니다.
- **자연어 프롬프트 입력:** 캔버스 상단 입력창에 찾고 싶은 물체(`helmet`, `head`, `person` 등)를 입력하면 AI가 자동으로 바운딩 박스를 그려줍니다.
- **민감도(Threshold) 조절:** 탐지 민감도를 상/중/하 3단계로 조절하여 오탐지(FP)와 미탐지(FN)의 균형을 제어합니다.
- **추론 결과 캐싱:** 한 번 수행한 AI 추론 결과는 `server/data/predictions/`에 JSON으로 자동 저장되어, 동일 이미지 재요청 시 모델 재실행 없이 즉시 반환됩니다.

### 6. 📊 성능 분석 대시보드 (Performance Analytics Dashboard) `NEW`
- **`GET /api/performance-metrics` 엔드포인트:** 저장된 정답 라벨(Ground Truth)과 AI 추론 결과를 비교하여 정량적 성능 지표를 계산하는 REST API를 제공합니다.
- **Class-Agnostic IoU 매칭:** 박스의 위치 겹침(IoU ≥ 0.5) 기반으로 TP/FP/FN을 계산하여 클래스 이름 불일치로 인한 오평가를 방지합니다.
- **Plotly.js 인터랙티브 차트:** 클래스별 Precision/Recall 막대 그래프 + IoU 분포 히스토그램을 웹 대시보드에서 실시간 시각화합니다.
  - 마우스 호버 시 클래스별 상세 지표 툴팁 표시
  - 범례(Legend) 클릭으로 특정 클래스만 필터링
- **계산 지표:**
  - `Precision` = TP / (TP + FP)
  - `Recall` = TP / (TP + FN)
  - `Average IoU` — 매칭된 박스들의 위치 정확도
  - `mAP` — 전체 클래스 평균 정밀도

### 7. 📥 성능 리포트 내보내기 (Report Export) `NEW`
- **[Download Report] 버튼:** 대시보드 상단 버튼 클릭 한 번으로 모든 성능 지표를 `performance_report.json` 파일로 즉시 다운로드합니다.
- **포함 데이터:** 클래스별 IoU, mAP, Precision, Recall, TP/FP/FN, 전체 집계 통계, IoU 분포 데이터 등 포트폴리오 및 논문 첨부에 활용 가능한 완전한 형태의 JSON 리포트를 제공합니다.

### 8. 🖥️ CLI 성능 평가 스크립트 (Local CLI Evaluation) `NEW`
- **`node cli_evaluate.js`:** 웹 브라우저 없이 터미널에서 직접 성능 평가를 실행할 수 있는 독립형 CLI 스크립트입니다.
- **표 형식 출력:** 평가 완료 시 클래스별 지표를 터미널에 보기 좋은 표(Table) 형태로 출력합니다.
- **클래스 인수 지원:** `node cli_evaluate.js "helmet,head"` 처럼 클래스를 동적으로 지정하여 CI/CD 자동화 파이프라인에 통합할 수 있습니다.

## 🎨 UI/UX 디자인 시스템 (M3 Based)

작업자의 눈 피로도를 최소화하고, 작업 영역의 가시성을 극대화하기 위해 **다크 모드(Dark Mode)** 기반의 M3 디자인 시스템을 적용했습니다.

### 디자인 원칙
- **Focus First:** 작업 영역(이미지 캔버스)의 시각적 노이즈 최소화
- **Efficient Interaction:** 마우스 이동 경로 최적화를 통한 3단 레이아웃 내 작업 완결
- **Contextual Clarity:** 현재 상태를 즉각적으로 알 수 있는 고대비 컬러 피드백

### 3단 레이아웃 구성
1. **좌측 (Sidebar):** 이미지 리스트 및 라벨링 완료 상태 표시 (`FileList`)
2. **중앙 (Canvas):** 마우스 드래그 박스 툴 및 정규화 좌표 실시간 피드백 오버레이 (`InteractionCanvas`)
3. **우측 (Panel):** 클래스 선택, 라벨 리스트 아이템 수정/삭제, 단축키 가이드 (`ClassController`)

## 🛠 기술 스택 (Tech Stack)

로컬 파일 처리 속도와 개발 생산성에 최적화된 아키텍처를 채택했습니다. 별도의 데이터베이스 없이 로컬 파일 시스템을 직접 제어하여 무결성과 속도를 확보했습니다.

### Frontend
- **React.js (Vite):** 빠르고 부드러운 UI 렌더링 및 인터랙티브 라벨링 로직 구현
- **Tailwind CSS:** 디자인 시스템을 반영한 빠르고 직관적인 UI 스타일링
- **Zustand:** 이미지 상태, 바운딩 박스 리스트 등 전역 상태의 효율적 관리
- **Plotly.js (CDN):** 성능 분석 대시보드의 인터랙티브 차트 시각화 (막대 그래프, 히스토그램)

### Backend (Local Bridge)
- **Node.js (Express.js):** 브라우저가 직접 접근할 수 없는 로컬 파일 시스템과의 브릿지 역할 담당
- **`@xenova/transformers`:** Hugging Face의 OWL-ViT 모델을 Node.js 환경에서 직접 실행하는 Zero-Shot 추론 엔진
- **`fs` 모듈:** `.txt` 라벨 파일 및 추론 결과 캐시 JSON 파일의 빠른 생성, 읽기, 쓰기

## 🚀 실행 방법

```bash
# 1. 백엔드 서버 실행 (server 폴더에서)
cd server
node server.js

# 2. 프론트엔드 개발 서버 실행 (client 폴더에서, 별도 터미널)
cd client
npm run dev

# 3. 브라우저에서 접속
# http://localhost:5173

# [선택] CLI로 성능 평가만 바로 실행
cd server
node cli_evaluate.js              # 기본: helmet, head, person
node cli_evaluate.js "car,truck"  # 커스텀 클래스 지정
```
